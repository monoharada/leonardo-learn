# Adobe Color 調査レポート

**調査日**: 2026年1月5日
**調査対象**: https://color.adobe.com/ja/create/color-wheel
**目的**: leonardo-learnのアクセントカラー選択機能改善のための競合分析

---

## 1. エグゼクティブサマリー

### 現状の課題
leonardo-learnの補色選択機能は「ブランドカラーの補色がグラデーションするだけ」という問題がある。Adobe Colorは同じ補色ハーモニーでも、より豊かな色のバリエーションを生成している。

### 主な発見
1. Adobe Colorは**9種類**のハーモニータイプを提供（leonardo-learnのアクセント選択UIは4種類）
2. 補色パレットは単純な色相反転ではなく、**明度・彩度のバリエーション**を含む
3. **モノクロマティック**、**コンパウンド**、**シェード**の3タイプがleonardo-learnのアクセント選択UIに未実装
4. パレットの構成に明確な**役割分担**がある（ベース、補色、アクセント等）

> **注**: leonardo-learnのコアモジュール（`HarmonyType` enum）にはSQUARE/TETRADIC等が定義されているが、アクセント選択UI（`HarmonyFilterType`）では4種類+allのみ提供。本レポートはUI機能の比較に焦点を当てる。

## 2. ハーモニータイプ一覧

### Adobe Colorで提供されている9タイプ

| # | 日本語名 | 英語名 | アクセント選択UI | コアモジュール | 特徴 |
|---|----------|--------|------------------|----------------|------|
| 1 | 類似色 | Analogous | ✅ | ✅ | 色相が隣接する色の組み合わせ |
| 2 | モノクロマティック | Monochromatic | ❌ | ❌ | 同一色相で彩度・明度を変化 |
| 3 | トライアド | Triad | ✅ | ✅ | 色相環を3等分（120度間隔） |
| 4 | 補色 | Complementary | ✅ | ✅ | 色相環の概ね反対側 |
| 5 | 分割補色 | Split-Complementary | ✅ | ✅ | 補色の両隣を使用 |
| 6 | 正方形 | Square | ❌ | ✅ | 色相環を4等分（90度間隔） |
| 7 | コンパウンド | Compound | ❌ | ❌ | 補色 + 類似色のハイブリッド |
| 8 | シェード | Shades | ❌ | ❌ | 同一色相で明度を変化 |
| 9 | カスタム | Custom | - | - | ユーザー自由設定 |

### leonardo-learnに不足している3タイプの詳細

#### モノクロマティック（Monochromatic）
- **定義**: 同じ色相（Hue）を維持しながら、彩度（Chroma）と明度（Lightness）を変化させる
- **視覚的特徴**: カラーホイール上で同じ角度の直線上に全ての色が配置される
- **用途**: ブランドカラーを基調とした統一感のあるUI、モノトーンデザイン

#### コンパウンド（Compound）
- **定義**: 補色と類似色を組み合わせたハイブリッドハーモニー
- **視覚的特徴**: ベース色の類似色 + 補色側の色を含む
- **用途**: 調和を保ちながらもコントラストを出したいデザイン

#### シェード（Shades）
- **定義**: 同じ色相・彩度を維持し、明度のみを変化させる
- **視覚的特徴**: カラーホイール上で1点に集中（同じ位置）
- **用途**: 単色のグラデーション、影の表現、階層的なUI要素

## 3. パレット構成の詳細分析

### 3.1 補色（Complementary）パレット

**テストカラー**: シアン系 #36F4D6

| 位置 | HEX | 役割 | 色相関係 |
|------|-----|------|----------|
| 1 | #F56E36 | 補色（メイン） | 概ね反対側（HSL: 約207度差） |
| 2 | #55B5A7 | ベース隣接（暗） | ベースの低彩度・低明度 |
| 3 | #36F4D6 | ベースカラー | 基準色（HSL H=170°） |
| 4 | #A06D58 | 補色（暗） | 補色の低彩度・低明度 |
| 5 | #547570 | ベース隣接（グレー） | ベースの低彩度 |

> **色相差の実測値**: ベース#36F4D6（HSL H=170.5°）と補色#F56E36（HSL H=17.6°）の色相差は約207度。Adobe Colorの「補色」は厳密な180度反転ではなく、視覚的な補色関係を重視している可能性がある。

**発見**:
- 補色パレットは単純な「ベース + 厳密な180度反転」ではない
- ベース側と補色側それぞれに**明度・彩度のバリエーション**を含む
- 5色中、補色関係にあるのは1色のみ、残りはトーンバリエーション

### 3.2 モノクロマティック（Monochromatic）パレット

**テストカラー**: シアン系 #36F5D8

| 位置 | HEX | 明度 | 彩度 |
|------|-----|------|------|
| 1 | #C0FAF0 | 高 | 低 |
| 2 | #7DF8E5 | 中高 | 中 |
| 3 | #36F5D8 | 中 | 高（ベース） |
| 4 | #2DCAB3 | 中低 | 中高 |
| 5 | #25A895 | 低 | 中 |

**発見**:
- 色相はほぼ同一（OKLCH H: 179〜183度、±4度以内の誤差）
- 明度と彩度が**反比例的に変化**する傾向
- 高明度 = 低彩度、低明度 = 中彩度のパターン

### 3.3 コンパウンド（Compound）パレット

**テストカラー**: シアン系 #36F5D8

| 位置 | HEX | 色相関係 | 役割 |
|------|-----|----------|------|
| 1 | #36F56C | 類似色（緑方向） | アクセント1 |
| 2 | #F5A536 | 補色隣接（黄橙） | アクセント2 |
| 3 | #36F5D8 | ベースカラー | 基準色 |
| 4 | #F56E36 | 補色（橙） | 補色メイン |
| 5 | - | - | - |

**発見**:
- ベースの**類似色**（±30度程度）と**補色側**の色を組み合わせ
- 補色単体よりも調和が取れつつ、コントラストも確保
- 「分割補色」に近いが、ベース側にも類似色を含む点が異なる

### 3.4 シェード（Shades）パレット

**テストカラー**: シアン系 #36F5D8

| 位置 | HEX | OKLCH L | OKLCH C | OKLCH H |
|------|-----|---------|---------|---------|
| 1 | #34ECD1 | 0.85 | 0.145 | 180° |
| 2 | #2DCAB3 | 0.76 | 0.128 | 180° |
| 3 | #36F5D8 | 0.87 | 0.150 | 179° |
| 4 | #25A895 | 0.66 | 0.111 | 180° |
| 5 | #1E8677 | 0.56 | 0.093 | 181° |
| 6 | #166459 | 0.45 | 0.075 | 181° |

> **注**: Adobe Colorのシェードは6色構成。他のハーモニータイプ（5色）とは異なる。

**発見**:
- 色相はほぼ固定（OKLCH H: 179〜181度、±2度以内）
- 彩度（クロマ）も明度に連動して変化（L低下に伴いC低下）
- 明度が主な変化軸だが、彩度も段階的に減少
- ボタンのホバー/アクティブ状態、階層的なUI要素に最適

## 4. アルゴリズム推測

### 4.1 色相オフセット

| ハーモニー | オフセット角度（理論値） | 実測値 |
|------------|--------------------------|--------|
| 類似色 | ±30度 | - |
| トライアド | ±120度 | - |
| 補色 | 180度 | 約207度（HSL） |
| 分割補色 | 150度, 210度 | - |
| 正方形 | 90度, 180度, 270度 | - |
| コンパウンド | ±30度 + 150〜210度 | - |

> **注**: Adobe Colorは独自のカラーホイールを使用している可能性があり、理論値と実測値に差がある。

### 4.2 明度・彩度の変化パターン

**補色パレットの場合**:
```
ベースカラー (H, S, L)
├── ベース暗: (H, S-20%, L-20%)
├── ベースグレー: (H, S-40%, L-10%)
├── 補色: (H+180, S, L)
└── 補色暗: (H+180, S-30%, L-30%)
```

**モノクロマティックの場合**:
```
ベースカラー (H, S, L)
├── 明1: (H, S-30%, L+30%)
├── 明2: (H, S-15%, L+15%)
├── 暗1: (H, S-10%, L-15%)
└── 暗2: (H, S-20%, L-30%)
```

## 5. leonardo-learnとの比較

### 5.1 現状の問題点

| 項目 | Adobe Color | leonardo-learn（アクセント選択UI） | 差分 |
|------|-------------|-----------------------------------|------|
| ハーモニータイプ数 | 9種類 | 4種類（+all） | -5種類 |
| パレット色数 | 5〜6色 | 3色 | 構成の違い |
| 補色パレット構成 | 明度・彩度バリエーション含む | グラデーション的 | 多様性不足 |
| 色の役割定義 | 明確（ベース、補色、アクセント） | 不明確 | 設計思想の差 |
| カスタマイズ性 | 高い（ドラッグで調整可能） | 限定的 | UX改善余地 |

### 5.2 根本的な違い

**Adobe Color**:
- 5〜6色パレットを**1つの調和した構成**として設計
- 各色に**機能的な役割**がある（メイン、サブ、アクセント等）
- 明度・彩度のバリエーションで**実用的なパレット**を生成

**leonardo-learn（現状）**:
- 色相回転による**単純な補色生成**
- 生成後の色は**等価な存在**（役割分担なし）
- グラデーション的な出力で**実用性に欠ける**

## 6. 改善提案

### 6.1 優先度：高

#### 新ハーモニータイプの追加
1. **モノクロマティック**: 実装が最も容易、需要も高い
2. **シェード**: ボタン状態やUI階層に有用
3. **コンパウンド**: 補色の改良版として価値が高い

#### 補色パレット生成アルゴリズムの改善
現在のグラデーション方式から、以下の構成に変更:
```
5色パレット = [
  ベース暗バリエーション,
  ベースカラー,
  ベース明バリエーション,
  補色,
  補色バリエーション
]
```

### 6.2 優先度：中

#### 色の役割定義の導入
```typescript
interface PaletteRole {
  base: Color;         // ブランドカラー
  baseLight: Color;    // ベースの明るいバリエーション
  baseDark: Color;     // ベースの暗いバリエーション
  accent: Color;       // アクセント（補色等）
  accentAlt: Color;    // アクセントのバリエーション
}
```

#### 正方形ハーモニーの追加
- 4色構成で、色相環を均等に4分割
- バランスの取れた多色パレットに有用

### 6.3 優先度：低

#### カスタムハーモニー機能
- ユーザーが色相オフセットを自由に設定
- 上級者向けの機能として検討

## 7. 技術的な実装方針（参考）

### OKLCH空間での実装

```typescript
// モノクロマティック生成の例
function generateMonochromatic(base: Oklch): Oklch[] {
  const variations = [
    { l: +0.25, c: -0.08 },  // 明るく、彩度低
    { l: +0.12, c: -0.04 },  // やや明るく
    { l: 0, c: 0 },          // ベース
    { l: -0.12, c: -0.02 },  // やや暗く
    { l: -0.25, c: -0.05 },  // 暗く
  ];

  return variations.map(v => ({
    mode: 'oklch',
    l: clamp(base.l + v.l, 0, 1),
    c: clamp(base.c + v.c, 0, 0.4),
    h: base.h  // 色相は固定
  }));
}

// シェード生成の例
function generateShades(base: Oklch): Oklch[] {
  const lightnessSteps = [0.85, 0.70, base.l, 0.45, 0.30, 0.20];

  return lightnessSteps.map(l => ({
    mode: 'oklch',
    l,
    c: base.c,  // 彩度固定
    h: base.h   // 色相固定
  }));
}
```

## 8. 結論

Adobe Colorの調査により、leonardo-learnのアクセント選択UI改善点が明確になった:

1. **ハーモニータイプの不足**: アクセント選択UIは4種類（コアモジュールにはSQUARE等も存在）
2. **パレット構成の単純さ**: 明度・彩度バリエーションがない（3色構成 vs Adobe Color 5〜6色）
3. **色の役割定義の欠如**: 各色の用途が不明確

これらの問題を解決することで、より実用的で魅力的なカラーパレット生成ツールとなる可能性がある。
