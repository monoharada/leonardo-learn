# Adobe Color 調査レポート

**調査日**: 2026年1月5日
**調査対象**: https://color.adobe.com/ja/create/color-wheel
**目的**: leonardo-learnのアクセントカラー選択機能改善のための競合分析

---

## 1. エグゼクティブサマリー

### 現状の課題
leonardo-learnの補色選択機能は「ブランドカラーの補色がグラデーションするだけ」という問題がある。Adobe Colorは同じ補色ハーモニーでも、より豊かな色のバリエーションを生成している。

### 主な発見
1. Adobe Colorは**9種類**のハーモニータイプを提供（leonardo-learnは4種類）
2. 補色パレットは単純な色相180度反転ではなく、**明度・彩度のバリエーション**を含む
3. **モノクロマティック**、**コンパウンド**、**シェード**の3タイプがleonardo-learnに未実装
4. 5色パレットの構成に明確な**役割分担**がある（ベース、補色、アクセント等）

---

## 2. ハーモニータイプ一覧

### Adobe Colorで提供されている9タイプ

| # | 日本語名 | 英語名 | leonardo-learn | 特徴 |
|---|----------|--------|----------------|------|
| 1 | 類似色 | Analogous | ✅ 実装済み | 色相が隣接する色の組み合わせ |
| 2 | モノクロマティック | Monochromatic | ❌ 未実装 | 同一色相で彩度・明度を変化 |
| 3 | トライアド | Triad | ✅ 実装済み | 色相環を3等分（120度間隔） |
| 4 | 補色 | Complementary | ✅ 実装済み | 色相環の反対側（180度） |
| 5 | 分割補色 | Split-Complementary | ✅ 実装済み | 補色の両隣を使用 |
| 6 | 正方形 | Square | ❌ 未実装 | 色相環を4等分（90度間隔） |
| 7 | コンパウンド | Compound | ❌ 未実装 | 補色 + 類似色のハイブリッド |
| 8 | シェード | Shades | ❌ 未実装 | 同一色相・彩度で明度のみ変化 |
| 9 | カスタム | Custom | - | ユーザー自由設定 |

### leonardo-learnに不足している3タイプの詳細

#### モノクロマティック（Monochromatic）
- **定義**: 同じ色相（Hue）を維持しながら、彩度（Chroma）と明度（Lightness）を変化させる
- **視覚的特徴**: カラーホイール上で同じ角度の直線上に全ての色が配置される
- **用途**: ブランドカラーを基調とした統一感のあるUI、モノトーンデザイン

#### コンパウンド（Compound）
- **定義**: 補色と類似色を組み合わせたハイブリッドハーモニー
- **視覚的特徴**: ベース色の類似色 + 補色側の色を含む
- **用途**: 調和を保ちながらもコントラストを出したいデザイン

#### シェード（Shades）
- **定義**: 同じ色相・彩度を維持し、明度のみを変化させる
- **視覚的特徴**: カラーホイール上で1点に集中（同じ位置）
- **用途**: 単色のグラデーション、影の表現、階層的なUI要素

## 3. パレット構成の詳細分析

### 3.1 補色（Complementary）パレット

**テストカラー**: シアン系 #36F4D6

| 位置 | HEX | 役割 | 色相関係 |
|------|-----|------|----------|
| 1 | #F56E36 | 補色（メイン） | 180度反転 |
| 2 | #55B5A7 | ベース隣接（暗） | ベースの低彩度・低明度 |
| 3 | #36F4D6 | ベースカラー | 基準色 |
| 4 | #A06D58 | 補色（暗） | 補色の低彩度・低明度 |
| 5 | #547570 | ベース隣接（グレー） | ベースの低彩度 |

**発見**:
- 補色パレットは単純な「ベース + 180度反転」ではない
- ベース側と補色側それぞれに**明度・彩度のバリエーション**を含む
- 5色中、純粋な補色は1色のみ、残りはトーンバリエーション

### 3.2 モノクロマティック（Monochromatic）パレット

**テストカラー**: シアン系 #36F5D8

| 位置 | HEX | 明度 | 彩度 |
|------|-----|------|------|
| 1 | #C0FAF0 | 高 | 低 |
| 2 | #7DF8E5 | 中高 | 中 |
| 3 | #36F5D8 | 中 | 高（ベース） |
| 4 | #2DCAB3 | 中低 | 中高 |
| 5 | #25A895 | 低 | 中 |

**発見**:
- 色相は完全に同一
- 明度と彩度が**反比例的に変化**する傾向
- 高明度 = 低彩度、低明度 = 中彩度のパターン

### 3.3 コンパウンド（Compound）パレット

**テストカラー**: シアン系 #36F5D8

| 位置 | HEX | 色相関係 | 役割 |
|------|-----|----------|------|
| 1 | #36F56C | 類似色（緑方向） | アクセント1 |
| 2 | #F5A536 | 補色隣接（黄橙） | アクセント2 |
| 3 | #36F5D8 | ベースカラー | 基準色 |
| 4 | #F56E36 | 補色（橙） | 補色メイン |
| 5 | - | - | - |

**発見**:
- ベースの**類似色**（±30度程度）と**補色側**の色を組み合わせ
- 補色単体よりも調和が取れつつ、コントラストも確保
- 「分割補色」に近いが、ベース側にも類似色を含む点が異なる

### 3.4 シェード（Shades）パレット

**テストカラー**: シアン系 #36F5D8

| 位置 | HEX | 明度変化 |
|------|-----|----------|
| 1 | #34ECD1 | +5% |
| 2 | #2DCAB3 | -10% |
| 3 | #36F5D8 | ベース |
| 4 | #25A895 | -25% |
| 5 | #1E8677 | -35% |
| 6 | #166459 | -45% |

**発見**:
- 色相・彩度は**完全固定**
- 明度のみが段階的に変化
- ボタンのホバー/アクティブ状態、階層的なUI要素に最適

## 4. アルゴリズム推測

### 4.1 色相オフセット

| ハーモニー | オフセット角度 |
|------------|---------------|
| 類似色 | ±30度 |
| トライアド | ±120度 |
| 補色 | 180度 |
| 分割補色 | 150度, 210度（±30度の補色） |
| 正方形 | 90度, 180度, 270度 |
| コンパウンド | ±30度（類似）+ 150〜210度（補色側） |

### 4.2 明度・彩度の変化パターン

**補色パレットの場合**:
```
ベースカラー (H, S, L)
├── ベース暗: (H, S-20%, L-20%)
├── ベースグレー: (H, S-40%, L-10%)
├── 補色: (H+180, S, L)
└── 補色暗: (H+180, S-30%, L-30%)
```

**モノクロマティックの場合**:
```
ベースカラー (H, S, L)
├── 明1: (H, S-30%, L+30%)
├── 明2: (H, S-15%, L+15%)
├── 暗1: (H, S-10%, L-15%)
└── 暗2: (H, S-20%, L-30%)
```

## 5. leonardo-learnとの比較

### 5.1 現状の問題点

| 項目 | Adobe Color | leonardo-learn | 差分 |
|------|-------------|----------------|------|
| ハーモニータイプ数 | 9種類 | 4種類 | -5種類 |
| 補色パレット構成 | 5色（明度・彩度バリエーション含む） | グラデーションのみ | 多様性不足 |
| 色の役割定義 | 明確（ベース、補色、アクセント） | 不明確 | 設計思想の差 |
| カスタマイズ性 | 高い（ドラッグで調整可能） | 限定的 | UX改善余地 |

### 5.2 根本的な違い

**Adobe Color**:
- 5色パレットを**1つの調和した構成**として設計
- 各色に**機能的な役割**がある（メイン、サブ、アクセント等）
- 明度・彩度のバリエーションで**実用的なパレット**を生成

**leonardo-learn（現状）**:
- 色相回転による**単純な補色生成**
- 生成後の色は**等価な存在**（役割分担なし）
- グラデーション的な出力で**実用性に欠ける**

## 6. 改善提案

### 6.1 優先度：高

#### 新ハーモニータイプの追加
1. **モノクロマティック**: 実装が最も容易、需要も高い
2. **シェード**: ボタン状態やUI階層に有用
3. **コンパウンド**: 補色の改良版として価値が高い

#### 補色パレット生成アルゴリズムの改善
現在のグラデーション方式から、以下の構成に変更:
```
5色パレット = [
  ベース暗バリエーション,
  ベースカラー,
  ベース明バリエーション,
  補色,
  補色バリエーション
]
```

### 6.2 優先度：中

#### 色の役割定義の導入
```typescript
interface PaletteRole {
  base: Color;         // ブランドカラー
  baseLight: Color;    // ベースの明るいバリエーション
  baseDark: Color;     // ベースの暗いバリエーション
  accent: Color;       // アクセント（補色等）
  accentAlt: Color;    // アクセントのバリエーション
}
```

#### 正方形ハーモニーの追加
- 4色構成で、色相環を均等に4分割
- バランスの取れた多色パレットに有用

### 6.3 優先度：低

#### カスタムハーモニー機能
- ユーザーが色相オフセットを自由に設定
- 上級者向けの機能として検討

## 7. 技術的な実装方針（参考）

### OKLCH空間での実装

```typescript
// モノクロマティック生成の例
function generateMonochromatic(base: Oklch): Oklch[] {
  const variations = [
    { l: +0.25, c: -0.08 },  // 明るく、彩度低
    { l: +0.12, c: -0.04 },  // やや明るく
    { l: 0, c: 0 },          // ベース
    { l: -0.12, c: -0.02 },  // やや暗く
    { l: -0.25, c: -0.05 },  // 暗く
  ];

  return variations.map(v => ({
    mode: 'oklch',
    l: clamp(base.l + v.l, 0, 1),
    c: clamp(base.c + v.c, 0, 0.4),
    h: base.h  // 色相は固定
  }));
}

// シェード生成の例
function generateShades(base: Oklch): Oklch[] {
  const lightnessSteps = [0.85, 0.70, base.l, 0.45, 0.30, 0.20];

  return lightnessSteps.map(l => ({
    mode: 'oklch',
    l,
    c: base.c,  // 彩度固定
    h: base.h   // 色相固定
  }));
}
```

## 8. 結論

Adobe Colorの調査により、leonardo-learnの改善点が明確になった:

1. **ハーモニータイプの不足**: 9種類中4種類のみ実装
2. **パレット構成の単純さ**: 明度・彩度バリエーションがない
3. **色の役割定義の欠如**: 各色の用途が不明確

これらの問題を解決することで、より実用的で魅力的なカラーパレット生成ツールとなる可能性がある。
