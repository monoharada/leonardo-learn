## 評価ルーブリック（Practical UI + DADS優先）
- 何の塊か：見出し・余白・面でグルーピングが読めるか（AP-H2）
- 何が押せるか：リンク/ボタン/非活性が区別できるか（AP-INT）
- 押したら何が起きるか：遷移/選択/送信の型が混ざっていないか（AP-INT）
- 状態の分かりやすさ：色だけに頼らず形・太字・位置で示せているか
- 色の役割：アクセント色が「リンク/CTA/状態/装飾」で兼務しすぎないか（AP-C2）
- a11y最小：フォーカス可視性、クリック領域、hover依存の有無

## 共通評価（フルページプレビュー系 00/02/04/10–18）
**良い点**
- セクション見出しがあり、ページ全体の段落構成は読み取りやすい。
- 主要CTAが1つに見えるため、初動の迷いは少ない。

**課題（共通）**
- 右カラムのリストが「入力フィールド/選択UI」に見えやすく、**遷移なのか選択なのか不明瞭**（AP-INT）。
- 「手続き案内」タイルは**選択状態と遷移の型が混在**して見える可能性（AP-INT）。
- カード/タイル/タグが同一アクセントを共有しやすく、**アクセント色が状態表現まで兼務**すると意味が重複しがち（AP-C2）。
- 角丸カードで `overflow: hidden` を使う場合、**focus-visible が切れるリスク**がある（静止画のため要確認）。

**改善の方向性（共通）**
- 遷移リストなら「行全体クリック + 右矢印」で統一（Practical UI レシピ準拠）。
- 状態表現は「左バー + 太字 + 形」を基本にし、色は補助に留める。
- タイルは「遷移」か「選択」かをUI形状で明確化（ラジオ風の丸は避ける）。

## アンチパターン対応表（要約）
| パターン | 該当傾向 | 主な該当画面 |
| --- | --- | --- |
| AP-INT（型の混在） | 遷移/選択/入力の見た目が近い | 00, 01, 03, 17 |
| AP-H1（重要度が読めない） | 重要要素が複数強調される | 01, 11, 12, 13, 15, 16, 18 |
| AP-H2（グルーピング不足） | 面や区切りが弱い/曖昧 | 14（ヘッダー/ヒーロー境界） |
| AP-C2（色の意味が重複） | アクセント色が状態/装飾/リンクを兼務 | 00/02/03/10-18（共通リスク） |

## 共通テーマ（横断）
1. **遷移UIの型を統一したい**  
   右カラム/タイル/チップが「入力」や「選択」に見えやすく、遷移導線が曖昧。
2. **CTAの主導権がビジュアルに押される**  
   Branding系は装飾面積が大きく、CTAと同等に見える。
3. **状態表現が色依存に寄りやすい**  
   ランダムカラーのため、色以外の形（バー/太字/配置）での補強が必要。

## 改善優先度（P0/P1/P2）
**P0（誤操作/迷いの削減）**
- 遷移リストは「行全体クリック + 右矢印」へ統一（AP-INT対策）
- タイルの「選択UI風」表現を排除し、遷移の型に寄せる

**P1（重要度・主従の整理）**
- Branding画面は**CTAを最上位の視線誘導**に固定（装飾は一段下げる）
- 複数強調の抑制（見出し/CTA/装飾の同時強調を避ける）

**P2（補助的改善）**
- 状態表現に「左バー + 太字 + 位置」の一貫ルールを追加
- focus-visibleのクリップ有無を実機で確認（角丸/overflow要注意）

## まとめ
- 最優先は**遷移UIの型統一**と**CTAの主導権維持**。
- ランダムカラー前提でも、**意味は形で統一**すれば評価は安定する。
