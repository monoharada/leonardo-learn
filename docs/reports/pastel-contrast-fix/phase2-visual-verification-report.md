## 4. 問題の根本原因（視覚的確認による仮説補強）

### 仮説の裏付け
Phase 1の数値検証で示された仮説が視覚的に確認された:

1. **パステル色の定義 (L ≥ 0.75, C ≤ 0.1)** により:
   - 生成される色が全て非常に明るい
   - 白または明るい背景に対してコントラストが確保できない

2. **フォールバックロジックの問題**:
   - コントラスト3:1を満たす色がない場合、条件を無視して色を選択
   - 結果として読めないUI要素が生成される

---

## 5. 問題が顕著なUI要素（優先度順）

1. **ヒーロー見出し** - 最も目立つ要素で最悪のコントラスト
2. **ナビゲーションリンク** - 重要なナビゲーションが読みにくい
3. **コンテンツ内リンク** - 記事内のリンクが識別困難
4. **ボタン/タグ** - 一部のUI要素も低コントラスト

---

## 6. 結論

### パステルプリセット固有の問題
- 他の4プリセット（Default, Vibrant, Dark, High Contrast）は全て識別性スコア95-100を達成
- **パステルプリセットのみ**が85（Bランク、警告）で、視覚的にも明確なコントラスト問題を示す
- 問題はパステルプリセット固有であり、アルゴリズムの根本的な修正が必要

### 推奨対策（Phase 4で詳細検討）
視覚的検証の結果、以下の対策候補の優先順位を提案:

1. **候補C: フォールバックロジックの改善** (最優先)
   - コントラスト不足時に明度を自動調整する処理を追加

2. **候補A: 背景色の自動調整**
   - パステル色を使用する場合は、背景を暗くすることでコントラスト確保

3. **候補B: 明度制約の緩和**
   - L ≥ 0.75 → L ≥ 0.55 など、パステルでもコントラストが確保できる範囲に調整
