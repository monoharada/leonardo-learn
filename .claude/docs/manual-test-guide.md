# Manual Test Guide - Leonardo-Learn
## Claude for Chrome を使った人間的UIテスト

このガイドは、Claude for Chrome を使用した手動UIテストのための操作指示書です。自然言語でのUI操作により、セレクター問題を回避し、実際のユーザー体験に近いテストを実現します。

## ハイブリッドテスト戦略

### なぜマニュアルテストが必要か

**現在の課題**:
- `data-testid` 属性が実装されていない（[Issue参照](../../.github/ISSUE_TEMPLATE/data-testid-implementation.md)）
- Playwright E2Eテストが実際のアプリではなくカスタムテストページを使用
- 視覚的検証（色の正確性、レイアウト）は自動テストで困難

**Claude for Chrome の利点**:
- ✅ **セレクター不要** - 自然言語で「ブランドカラー入力欄」と指示可能
- ✅ **視覚的検証** - 「色が正しく表示されているか」確認可能
- ✅ **即座の実行** - コード修正不要、今すぐテスト開始
- ✅ **探索的テスト** - 予期しない問題の発見
- ✅ **エビデンス収集** - スクリーンショット・操作録画が容易

### 推奨テスト戦略

```
Phase 1: 探索的テスト（Manual - Claude for Chrome）
    ↓ 初期検証、問題発見、視覚的確認

Phase 2: 自動化テスト（Automated - Playwright E2E）
    ↓ リグレッション防止、CI/CD統合
```

**使い分け**:
- **マニュアル（本ガイド）**: 新機能追加時、UI変更時、視覚的検証、探索的テスト
- **自動化（Playwright）**: 継続的インテグレーション、リグレッションテスト、パフォーマンステスト

## Claude for Chrome 使用方法

### 準備

1. **Chrome拡張機能のインストール**:
   - [Claude for Chrome](https://claude.com/blog/claude-for-chrome) をインストール

2. **アプリケーションの起動**:
   ```bash
   cd /path/to/leonardo-learn
   bun run dev
   ```
   - ブラウザで `http://localhost:3000` を開く

3. **Claude for Chrome の起動**:
   - Chrome拡張機能アイコンをクリック
   - 本ガイドのテストシナリオをコピー&ペーストして実行

### 基本的な使い方

**指示の書き方**:
```
悪い例（セレクター依存）:
"#brand-color-input に #3B82F6 と入力してください"

良い例（自然言語）:
"ブランドカラー入力欄に #3B82F6 と入力してください"
```

**期待結果の確認**:
```
"パレットビューが表示されていることを確認してください"
"2色のカラースケールが見えますか？"
"各色に13段階のシェードがありますか？"
```

### よくある質問（FAQ）

**Q: Claude for Chrome が操作できない要素があったらどうすればいいですか？**

A: 以下の方法を試してください：
1. より具体的な指示（「画面左上の青いボタン」など）
2. スクリーンショットを撮って、要素を特定
3. 手動で操作して、結果のみ Claude に確認依頼
4. GitHub Issue で報告（自動化時に対処）

**Q: エラーが発生したらどうすればいいですか？**

A: 以下を記録してください：
1. エラーメッセージ（スクリーンショット）
2. 発生したステップ
3. ブラウザコンソールのエラー（F12 → Console）
4. 再現手順
5. `evidence/errors/` に保存

**Q: マニュアルテストの頻度は？**

A: 以下のタイミングで実施：
- 新機能追加時
- UIデザイン変更時
- リリース前（全シナリオ）
- 探索的テスト（週1回推奨）
