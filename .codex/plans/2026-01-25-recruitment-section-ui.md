# 採用情報セクション（採用種別リンクリスト＋イラストカード）UI改善（DADS準拠）

Date: 2026-01-25
Source: `.context/attachments/pasted_text_2026-01-25_10-40-03.txt`

## 目標
- 右の「採用種別」が **“リンクで遷移するリスト”** と即座に分かる（混乱の解消）
- 選択中（現在地）が **色以外でも** 分かる（形/太さ/バー等）
- DADSの `:focus-visible` が **見切れず** 明確に見える（アクセシビリティ）

## 背景
- 行政サービスサイトの2つのUI要素（左：カード、右：採用種別リスト）について、視覚的階層の曖昧さとインタラクション混在が課題。
- 特に、ラジオ風の●/○とナビゲーション風の→が混在し「選択なのか遷移なのか」が伝わらない。
- 紫系が「装飾」「リンク」「選択状態」の複数の意味で使われ、視線誘導が散る（AP-C2）。
- 実装面では `.preview-list` の `overflow: hidden` により、`.dads-link:focus-visible` のアウトラインがクリップされうる。

## スコープ
- やること：
  - 「採用種別」は **リンクリスト（遷移）** として統一（ラジオ的表現を排除）
  - 選択状態を「左アクセントバー + 太字」などで表現（色以外の差分を追加）
  - フォーカス見切れ（a11y）を解消
  - セクション見出しを追加して左右の関係性を明確化（AP-H2対策）
  - 「practical-ui-2nd-edition」スキル（テンプレ/ルール）を追加し、DADS優先で運用できる形にする
- やらないこと：
  - ラジオボタン機能（JSや状態管理の追加）
  - パレット生成/色計算ロジックの変更

## 前提 / 制約
- DADSのリンク/フォーカス原則（視認性・キーボード操作）を優先し、過剰な独自フォーカス表現は入れない
- `--preview-*` 変数ベースでテーマ/プリセット変更に耐える
- 変更は局所（two-columnセクション）に限定し、不要な整形は避ける

## 変更内容（案）
### データ / バックエンド
- 該当なし

### UI / UX
1) **右リストを “ナビゲーションリスト” に統一**
- ラジオボタン風の表現（●/○）を廃止
- `→` は維持し、リンクであることを明確化
- selected item に `aria-current="page"` を付与して「現在地」を支援技術にも伝える

2) **選択状態（現在地）の視認性を上げる（色以外の差分）**
- 左アクセントバー（2–4px相当） + 文字太さ/矢印色変化
- 背景はアクセント色ベタではなく、ニュートラル寄りの薄い面（AP-C2対策）

3) **フォーカス見切れ（a11y）を解消**
- `.preview-list` の `overflow: hidden` を見直し、DADS `:focus-visible` のアウトラインが見えるようにする
- 角丸維持は「先頭/末尾アイテムに角丸付与」等で担保

4) **左右の関係性（グルーピング）を強める**
- two-column の上に「採用情報」見出し（h2）を追加
- 「採用種別」は見出しレベルを下げて（h3）階層を整理（AP-H2対策）

### その他（Docs/Marketing/Infra など）
- 「practical-ui-2nd-edition」スキルを `.codex/skills/` に追加（DADS優先のチェックリスト化）

## 受入基準
- [ ] 採用種別リストが「選択」ではなく **遷移リンク**として見える（ラジオ風の丸が残っていない）
- [ ] selected（現在地）が **色以外**（左バー/太さ等）でも判別できる
- [ ] リスト内リンクにキーボードフォーカスしても **focus-visible が見切れない**
- [ ] セクション見出しにより、カード＋リストの関係性が分かる（AP-H2対策）
- [ ] テーマ/プリセット変更でも破綻しない（`--preview-*` 変数運用のまま）
- [ ] `bun test` がパスする

## リスク / エッジケース
- `overflow` 変更で角丸/罫線の見え方が崩れる（→先頭/末尾の角丸付与で回避）
- selected の強調を上げすぎるとテーマによっては主張が強くなりすぎる（→ mix率を小さく段階調整）
- 追加見出しで見出し階層が不自然になる（→ h2/h3 の整理で回避）
