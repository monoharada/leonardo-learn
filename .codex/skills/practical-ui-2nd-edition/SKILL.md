# practical-ui-2nd-edition（DADS優先）
# 公共サービス品質（DADS）を前提に、Practical UI（2nd）系の原則でUIを改善するためのスキル。
# ============================================

## 目的
- UI改善を「見た目」ではなく **認知負荷の最小化 / 階層の明確化 / 迷いの排除**として扱う。
- DADS（デジタル庁デザインシステム）準拠を最優先し、Practical UIの原則は **DADSの上に重ねる補助輪**として使う。

## 絶対ルール（DADS優先）
- **フォーカス可視性を壊さない**（`focus-visible` がクリップされない / 目立つ / 一貫している）。
- **意味は見た目で1つにする**（同じ色/形が「装飾」「リンク」「選択」など複数の意味を持たない）。
- **インタラクションの型を混ぜない**
  - 遷移（ナビ）: 「リンク/矢印/ホバー」で統一
  - 選択（ラジオ/チェック）: 「選択UI/確定行動」で統一
  - 迷ったら、まず「遷移リンク」に寄せる（公共サービスでは誤操作が高コスト）

## クイック診断（60秒）
1. **これは何の塊？**（見出しがあるか、余白でグループになっているか）
2. **何が押せる？**（リンク/ボタン/非活性が一目で区別できるか）
3. **押したら何が起きる？**（遷移/展開/選択/送信が混ざっていないか）
4. **現在地/状態が分かる？**（色だけに頼らず、形・太字・アイコンで分かるか）
5. **キーボードで迷わない？**（Tab順、フォーカスリング、クリック領域）

## よくあるアンチパターン（避ける）
- **AP-C2（色の意味が重複）**: 1つのアクセント色が「装飾」「リンク」「選択背景」まで兼務する
- **AP-H2（グルーピング不足）**: 並置されているのに関係性が読めない（見出し/余白/面が弱い）
- **AP-H1（重要度が読めない）**: すべて同じ強さで、次に何をすべきか決められない
- **AP-INT（型の混在）**: ラジオ風の●と遷移矢印→が同居する、など

## 改善レシピ
### 1) 「リンクリスト（遷移）」の整え方
- 左端の「丸（ラジオ風）」は使わない（誤認を生む）
- 行全体をクリック可能にし、右端に `→` を置く
- 状態（現在地）:
  - **左アクセントバー** + **太字** + **矢印色** で表現（色だけにしない）
  - 背景は派手にしない（ニュートラル面で十分）
- `overflow: hidden` でフォーカスリングを切らない（角丸は先頭/末尾の角丸で担保）

### 2) 2カラム（カード + リスト）の整え方
- まず **セクション見出し（h2）** を上に置き、2つの関係性を宣言する
- 左右の「面」を揃える（角丸/罫線/余白のルールを一致させる）
- 強調は1つだけ（「どこが一番重要か」を1秒で分かるようにする）

### 3) 色の役割整理（最小ルール）
- アクセント色は **リンク/CTA** に優先配分
- 選択/現在地は「形 + 太字 + ニュートラル面」を基本にし、色は補助にする
- 装飾（イラスト等）は彩度を上げてもよいが、**UI状態と同じ表現を使わない**

## 最低限のa11yチェック
- [ ] `focus-visible` が見切れない（親要素の `overflow`、隣接要素との重なり）
- [ ] 現在地/状態が色以外でも分かる（太字/バー/アイコン等）
- [ ] クリック領域が十分（行全体クリック、行間が詰まりすぎない）
- [ ] hoverは「任意」扱い（`@media (hover: hover)` で分岐）
